## Customization

MStepProgress is styled with CSS variables. See **Guides/Theming & Customization** for the
override mechanism (global / zonal / instance). Below is the component's public token API.

**Geometry — component tokens** (override per-component; default falls to a semantic role):

| Token | Controls | Default |
|-------|----------|---------|
| `--step-progress-padding-y` | Vertical padding of each step | `--gap-xs` |
| `--step-progress-padding-x` | Horizontal padding of each step | `--gap-s` |
| `--step-progress-radius` | Corner radius of each step | `--radius-xs` |

**Colors** have no per-component token by design — they bind to shared semantic roles so
MStepProgress stays in lock-step with the rest of the system.

| Mode | Semantic role | → primitive (default) |
|------|----------------|-----------------------|
| pending | `--color-surface-overlay` | `--palette-base-100` |
| info | `--color-status-info` / `-surface` / `-border` | `--palette-info-800` / `-100` / `-400` |
| warning | `--color-status-warning` / `-surface` / `-border` | `--palette-warning-800` / `-100` / `-400` |
| error | `--color-status-error` / `-surface` / `-border` | `--palette-error-800` / `-100` / `-400` |
| success | `--palette-success-600` / `--color-status-success-surface` | `--palette-success-600` / `--palette-success-100` |

> Recoloring for a whole brand is a global concern — see **Guides/Theming & Customization**.
