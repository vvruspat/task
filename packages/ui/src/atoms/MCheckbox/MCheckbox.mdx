## Customization

MCheckbox is styled with CSS variables. See **Guides/Theming & Customization** for the
override mechanism (global / zonal / instance). Below is the component's public token API.

**Geometry — component tokens** (override per-component; default falls to a semantic role):

| Token | Controls | Default |
|-------|----------|---------|
| `--checkbox-gap` | Gap between the box and the label | `--gap-m` |
| `--checkbox-size` | Box width/height | `--gap-l` |
| `--checkbox-radius` | Box corner radius | `--radius-xs` |

**Colors & focus** have no per-checkbox token by design — they bind to shared semantic
roles so the checkbox stays in lock-step with the rest of the system.

| Status | Semantic role | → primitive (default) |
|--------|---------------|-----------------------|
| regular · box border | `--color-control-outline` / `-hover` | `--palette-base-600` / `-700` |
| regular · checked | `--color-control-checked` / `-hover` | `--palette-primary-600` / `-500` |
| regular · icon | `--color-control-thumb` | `--palette-base-0` |
| valid | `--color-control-checked` | `--palette-primary-600` |
| invalid · border/icon | `--color-border-error` / `--color-status-error` | `--palette-error-600` / `-800` |
| disabled | `--color-action-disabled` | `--palette-base-300` |
| focus | `--color-focus-ring` | `--palette-primary-700` |

Focus geometry is the shared `--focus-ring-width` / `--focus-ring-offset`.

> Recoloring for a whole brand is a global concern — see **Guides/Theming & Customization**.
