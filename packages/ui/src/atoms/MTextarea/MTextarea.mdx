## Customization

MTextarea is styled with CSS variables. See **Guides/Theming & Customization** for the
override mechanism (global / zonal / instance). Below is the component's public token API.

**Geometry — component tokens** (override per-component; default falls to a semantic role):

| Token | Controls | Default |
|-------|----------|---------|
| `--textarea-gap` | Gap between label row / container / description | `--gap-m` |
| `--textarea-radius` | Container corner radius | `--radius-control-sm` · mobile/tablet: `--radius-control` |
| `--textarea-px` | Horizontal padding (also positions the counter) | `--padding-control-x-s` · mobile/tablet: `--padding-control-x-m` |
| `--textarea-py` | Vertical padding (also positions the counter) | `--padding-control-y-m` · mobile/tablet: `--gap-m` |
| `--textarea-counter-space` | Extra bottom padding reserved for the char counter | `--size-control-m` |

**Colors & focus** have no per-textarea token by design — they bind to shared semantic
roles so the textarea stays in lock-step with the rest of the system.

| Status | Semantic role | → primitive (default) |
|--------|---------------|-----------------------|
| regular | `--color-input-border` | `--palette-alpha-black-12` |
| regular | `--color-input-placeholder` / `-placeholder-hover` | `--palette-base-300` / `--palette-base-400` |
| valid | `--color-border-success` | `--palette-success-700` |
| invalid | `--color-border-error` / `--color-status-error` | `--palette-error-600` / `--palette-error-800` |
| disabled | `--color-input-bg-disabled` | `--palette-base-100` |
| focus ring | `--color-focus-ring` | `--palette-primary-700` |

Focus geometry is the shared `--focus-ring-width` / `--focus-ring-offset`.

> Recoloring for a whole brand is a global concern — see **Guides/Theming & Customization**.
