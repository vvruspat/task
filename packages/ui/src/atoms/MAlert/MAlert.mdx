A banner-style feedback component for communicating system messages. Supports four semantic modes corresponding to different message severities.

## Modes

| Mode | When to use |
|------|-------------|
| `info` | Neutral information — tips, hints, notices |
| `success` | Positive outcome — action completed |
| `warning` | Caution — potential issue, confirm before continuing |
| `error` | Failure — something went wrong, action required |

## Customization

MAlert is styled with CSS variables. See **Guides/Theming & Customization** for the
override mechanism (global / zonal / instance). Below is the component's public token API.

**Geometry — component tokens** (override per-component; default falls to a semantic role):

| Token | Controls | Default |
|-------|----------|---------|
| `--alert-radius` | Corner radius | `--radius-md` |
| `--alert-py` | Vertical padding | `--gap-s` |
| `--alert-px` | Horizontal padding | `--gap-l` |

**Colors** have no per-alert token by design — they bind to shared semantic status roles so
the alert stays in lock-step with the rest of the system. The table shows which role each
mode consumes and the primitive it resolves to in the default (light) theme.

| Mode | Semantic role | → primitive (default) |
|------|---------------|-----------------------|
| info | `--color-status-info` / `-surface` / `-border` | `--palette-info-800` / `-100` / `-400` |
| success | `--color-status-success` / `-surface` / `-border` | `--palette-success-800` / `-100` / `-400` |
| warning | `--color-status-warning` / `-surface` / `-border` | `--palette-warning-800` / `-100` / `-400` |
| error | `--color-status-error` / `-surface` / `-border` | `--palette-error-800` / `-100` / `-400` |

> Recoloring for a whole brand is a global concern — see **Guides/Theming & Customization**.
