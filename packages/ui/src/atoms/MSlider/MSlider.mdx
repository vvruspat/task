## Customization

MSlider is styled with CSS variables. See **Guides/Theming & Customization** for the
override mechanism (global / zonal / instance). Below is the component's public token API.

**Geometry — component tokens** (override per-component; default falls to a semantic role):

| Token | Controls | Default |
|-------|----------|---------|
| `--slider-gap` | Vertical gap between header, track, and description | `--gap-xs` |
| `--slider-header-gap` | Gap between label and value in the header row | `--gap-m` |
| `--slider-hit-height` | Min-height of the track's click/hit area | `--gap-xl` |
| `--slider-track-height` | Visual bar height | `--gap-xs` |
| `--slider-track-radius` | Bar and thumb corner radius | `--radius-pill` |
| `--slider-thumb-size` | Thumb width/height | `--gap-l` |
| `--slider-thumb-border-width` | Thumb border width | `--gap-3xs` |

**Colors & focus** have no per-slider token by design — they bind to shared semantic
roles so the slider stays in lock-step with the rest of the system.

| Status | Semantic role | → primitive (default) |
|--------|---------------|-----------------------|
| regular · track | `--color-control-track` | `--palette-base-200` |
| regular · fill/thumb border | `--color-control-checked` / `-hover` | `--palette-primary-600` / `-500` |
| regular · thumb | `--color-control-thumb` | `--palette-base-0` |
| valid | `--color-status-success-border` / `--color-status-success` | `--palette-success-400` / `-800` |
| invalid | `--color-status-error-border` / `--color-status-error` | `--palette-error-400` / `-800` |
| disabled | `--color-action-disabled` | `--palette-base-300` |
| focus | `--color-focus-ring` | `--palette-primary-700` |

Focus geometry is the shared `--focus-ring-width` / `--focus-ring-offset`.

> Recoloring for a whole brand is a global concern — see **Guides/Theming & Customization**.
