# Definition

Input is a clickable interactive element that triggers a response.

You can place text and icons inside of a Input.

Inputs are often used for form submissions and to toggle elements into view.

## Customization

MFileInput is styled with CSS variables. See **Guides/Theming & Customization** for the
override mechanism (global / zonal / instance). Below is the component's public token API.

**Geometry — component tokens** (override per-component; default falls to a semantic role):

| Token | Controls | Default |
|-------|----------|---------|
| `--fileinput-gap` | Gap between the file list text and the trigger button | `--gap-control` |
| `--fileinput-drop-radius` | Upload-modal drop-zone corner radius | `--radius-xl` |
| `--fileinput-drop-padding` | Upload-modal drop-zone padding | `--gap-4xl` |

**Colors & focus** have no per-component token by design — they bind to shared semantic
roles so the component stays in lock-step with the rest of the system.

| Part / status | Semantic role | → primitive (default) |
|----------------|---------------|-----------------------|
| regular | `--color-input-text` | `--palette-base-400` |
| valid | `--color-status-success` | `--palette-success-800` |
| invalid | `--color-status-error` | `--palette-error-800` |
| drop-zone focus ring | `--color-focus-ring` | `--palette-primary-700` |

Focus geometry is the shared `--focus-ring-width` / `--focus-ring-offset`. The trigger
button is an `MButton` and follows its own token contract — see **MButton → Customization**.

> Recoloring for a whole brand is a global concern — see **Guides/Theming & Customization**.
