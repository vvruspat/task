## Customization

MAccordion is styled with CSS variables. See **Guides/Theming & Customization** for the
override mechanism (global / zonal / instance). Below is the component's public token API.

**Geometry — component tokens** (override per-component; default falls to a semantic role):

| Token | Controls | Default |
|-------|----------|---------|
| `--accordion-header-padding-y` | Vertical padding of each item header | `--gap-m` |
| `--accordion-header-padding-x` | Horizontal padding of each item header | `--gap-l` |
| `--accordion-gap` | Gap between the title and the caret icon | `--gap-m` |
| `--accordion-panel-padding-x` | Horizontal padding of the expanded panel | `--gap-l` |
| `--accordion-panel-padding-bottom` | Bottom padding of the expanded panel | `--gap-l` |
| `--accordion-panel-content-padding-top` | Top padding inside the panel content | `--gap-xs` |

The outer surface (radius, shadow, border) comes from the `MCard` it's built on — see the
`MCard` Customization section for `--card-*` tokens.

**Colors** have no per-component token by design — they bind to shared semantic roles so
MAccordion stays in lock-step with the rest of the system.

| State | Semantic role | → primitive (default) |
|-------|----------------|-----------------------|
| item background | `--color-surface` | `--palette-base-0` |
| expanded item background | `--color-surface-raised` | `--palette-base-50` |
| header hover | `--color-surface-overlay` | `--palette-base-100` |
| header pressed | `--color-surface-sunken` | `--palette-base-200` |
| text | `--color-text-primary` / `--color-text-secondary` | `--palette-base-800` / `-600` |
| disabled text | `--color-text-disabled` | `--palette-base-300` |
| divider | `--color-border-default` | `--palette-alpha-black-12` |
| focus ring | `--color-focus-ring` | `--palette-primary-700` |

Focus geometry is the shared `--focus-ring-width` / `--focus-ring-offset`.

> Recoloring for a whole brand is a global concern — see **Guides/Theming & Customization**.
