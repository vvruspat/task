# Product model

## Что строим

tAsk - это task tracker для музыкальных и креативных проектов, где главным интерфейсом является Telegram-бот с LLM, а веб-приложение дает визуальные views: Kanban, таблицу как в Excel, настройки, task skills/templates, пользователей и файлы.

Цель - повторить ключевые преимущества Linear по скорости и ясности, но адаптировать систему под production-процессы: альбомы, песни, клипы, мюзиклы, релизы, репетиции и любые другие workflows.

## Принцип без жестких типов

В системе не должно быть жестких типов вроде:

```text
project.type = album
task.type = song
```

Вместо этого все строится из универсальных сущностей:

- Workspace / папка;
- Project;
- Task;
- Subtask;
- Task Skill / Template;
- User;
- Comment;
- Attachment;
- Status.

"Альбом", "песня", "клип", "инструментал", "сцена мюзикла" - это не типы в коде. Это названия проектов, задач и пользовательских task skills.

## Task skills/templates

Task skill - это пользовательский шаблон, который учит систему создавать дерево задач.

Пример:

```text
@task создай шаблон таски "песня" в нем должны быть сабтаски:
- Текст Rus
- Текст Eng
- Вокальная мелодия
- Аранжировка
- Партия ударных
- Бас
- Гитара ритм
- Гитара соло
- Запись вокала
- Сведение
- Мастеринг
```

Другой пример:

```text
@task создай шаблон "инструментал" с полями:
- Аранжировка
- Партия ударных
- Бас
```

После этого пользователь может написать:

```text
@task создай песню "моя песня 3" в альбоме "мой альбом"
```

Система должна:

1. Понять, какой workspace и Telegram-пользователь участвуют в запросе.
2. Найти проект с именем, похожим на "мой альбом".
3. Найти task skill, похожий на "песня".
4. Подготовить preview создания задачи "моя песня 3" внутри проекта.
5. Применить task skill "песня" и создать сабтаски.
6. Попросить подтверждение, если создается много объектов или есть неоднозначность.

## Изменение шаблонов под конкретную задачу

Шаблон не должен быть жестким. Пользователь может изменить его прямо в запросе:

```text
@task создай песню "Интро" в "мой альбом", без вокала и без гитар
```

Ожидаемое поведение:

- взять task skill "песня";
- убрать задачи, связанные с вокалом и гитарами;
- показать preview;
- после подтверждения создать только нужные сабтаски.

Другой пример:

```text
@task создай песню "финал", как обычную песню, но добавь хор и струнные
```

Система применяет skill "песня" с overrides:

- add: "Хор";
- add: "Струнные".

## Основные views

### Kanban

Linear-like доска:

- колонки по статусам;
- карточки задач;
- быстрые assignee, due date, labels, attachments;
- drag-and-drop;
- фильтры по проекту, человеку, label, дедлайну.

### Matrix view

View, похожий на текущую Excel-таблицу:

- колонки - parent tasks внутри проекта, например песни;
- строки - сабтаски/этапы;
- ячейки - статус, ответственный, комментарий или короткое значение;
- цвета статусов: нет, в процессе, готово, заблокировано;
- быстрый обзор прогресса всего альбома/проекта.

Matrix view важен как отдельная продуктовая фича, а не как импорт Excel.

### Table view

Плоский обзор всех задач:

- project;
- parent task;
- title;
- status;
- assignee;
- due date;
- labels;
- last update.

### My tasks

Персональный список:

- что назначено на пользователя;
- что просрочено;
- что ждет проверки;
- что было изменено недавно.

### Templates/skills editor

Интерфейс для создания и редактирования task skills:

- название;
- aliases;
- описание;
- список сабтасок;
- optional steps;
- вложенные сабтаски;
- default assignees;
- default descriptions;
- labels.

## Telegram-first UX

Telegram-бот должен уметь быть главным интерфейсом:

```text
@task что по альбому "мой альбом"?
@task добавь задачу записать бас в песню "моя песня 3"
@task отметь "Сведение" по "моя песня 3" как готово
@task прикрепи этот файл к песне "моя песня 3"
@task создай шаблон "релиз" с задачами ...
@task сделай summary по проекту за неделю
```

LLM отвечает коротко, но если действие меняет много данных, показывает preview:

```text
Создать задачу "моя песня 3" в проекте "мой альбом"?

Будут добавлены сабтаски:
- Текст Rus
- Текст Eng
- Вокальная мелодия
- Аранжировка
- Партия ударных
- Бас
- Гитара ритм
- Гитара соло
- Запись вокала
- Сведение
- Мастеринг

[Создать] [Изменить] [Отмена]
```

## Синхронизация пользователей Telegram

Главный пользователь находится внутри tAsk. Telegram - это identity provider и канал управления.

Не использовать Telegram username как стабильный ID. Username можно поменять. Основной ключ - `telegram_id`.

Поток привязки:

1. Админ создает пользователя в workspace или отправляет invite.
2. Система генерирует ссылку:

```text
https://t.me/task_bot?start=invite_<token>
```

3. Пользователь открывает ссылку в Telegram.
4. Бот получает `telegram_id`.
5. Backend связывает `telegram_id` с внутренним `user_id`.

Если пользователь пишет боту без привязки, бот должен предложить:

- ввести invite code;
- запросить доступ;
- открыть Mini App для привязки.
